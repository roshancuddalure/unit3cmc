# Tab Badge Status Pattern

## Purpose

Use this when a tab bar should do more than switch views and can safely communicate lightweight status at a glance.

## Good badge use cases

- count of recent items
- count of cases in current range
- count of members with activity
- short labels like `New`

## Rules

- Keep badges lightweight and glanceable.
- Use values the page already computes when possible.
- Do not overload the tab with dense explanatory text.
- Badge meaning should remain stable within the current page context.

## Why this works

A tab bar becomes more useful when it doubles as a compact status strip. Users can decide where to go next before opening each workspace panel.
